# pinchpass

One-time secret request links for OpenClaw agents. Generates a temporary HTTP server with a single-use encrypted form for collecting sensitive values вЂ” works both locally and over the internet via a built-in bore.pub tunnel.

**The relay never sees the secret.** Secrets are encrypted client-side in the browser before transmission using XSalsa20-Poly1305 (TweetNaCl). The encryption key lives only in the URL fragment вЂ” it is never sent to the server or the tunnel relay.

## How it works

```
Agent runs:   pinchpass request GEMINI_API_KEY --tunnel --note "Google AI Studio key"
              в†’ http://bore.pub:38448/claim/abc123вЂ¦#k=a1b2c3вЂ¦

User opens    в†’ form with embedded TweetNaCl.js crypto (no WebCrypto required)
User submits  в†’ browser encrypts value, POSTs ciphertext to server
Agent knows   в†’ decrypts locally, writes to .env вњ…
```

The link is:
- **One-time** вЂ” single use, invalidated after first submission
- **E2E encrypted** вЂ” key in URL fragment, never leaves the browser
- **Time-limited** вЂ” expires after TTL (default 30 min)
- **Token-authenticated** вЂ” 32-byte random hex token in the URL path
- **Works anywhere** вЂ” local or public via bore.pub tunnel (no install required)

## Usage

```bash
# Local link (LAN only)
pinchpass request GEMINI_API_KEY

# Public link via bore.pub tunnel
pinchpass request GEMINI_API_KEY --tunnel

# With note and custom output file
pinchpass request GEMINI_API_KEY --note "Google AI Studio key" --out config/secrets.env

# JSON output for agent parsing
pinchpass request GEMINI_API_KEY --json

# Custom TTL (minutes)
pinchpass request GEMINI_API_KEY --ttl 5
```

## Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--tunnel` | false | Open a bore.pub tunnel for a public URL |
| `--note` | вЂ” | Description shown on the form |
| `--out` | `.env` | Output file path |
| `--ttl` | 30 | Minutes until the link expires |
| `--port` | random | Local port to bind |
| `--json` | false | Machine-readable output |

## Output

The secret is saved to a `.env` file (default: `.env`):

```env
GEMINI_API_KEY="AIzaSy..."
```

The value is shell-escaped. Existing keys are overwritten in place.

## Security model

```
Browser                        Server (local)        Bore relay
  в”‚                                  в”‚                    в”‚
  в”‚  key = URL fragment (#k=вЂ¦)       в”‚                    в”‚
  в”‚  nonce = crypto.getRandomValues  в”‚                    в”‚
  в”‚  box = nacl.secretbox(val, вЂ¦)    в”‚                    в”‚
  в”‚                                  в”‚                    в”‚
  в”‚ в”Ђв”Ђ POST base64(nonce вЂ– box) в”Ђв”Ђв”Ђв–є в”‚ в”Ђв”Ђ ciphertext в”Ђв”Ђв–є в”‚
  в”‚                                  в”‚                    в”‚

  в”‚                                  в”‚
```

- Cipher: **XSalsa20-Poly1305** (TweetNaCl.js in browser, `golang.org/x/crypto/nacl/secretbox` in Go)
- Key: 32 random bytes generated by the CLI, delivered via URL fragment only
- Works on plain HTTP вЂ” no HTTPS or WebCrypto required

## OpenClaw plugin

This repo includes an OpenClaw plugin that registers a `request_secret` tool
for collecting secrets from users. It also bundles a skill that teaches agents
how to use `pinchpass` directly.

### Local install (auto-discovered)

The plugin at `.opencode/plugins/pinchpass/plugin.js` is auto-discovered when
OpenClaw runs in this repo. No config needed.

### Install from npm

```bash
openclaw plugins install @pinchpass/cli
```

### Manual install

```bash
cd plugins/pinchpass
npm install && npm run build
# Copy the skill to your skills directory
cp SKILL.md ~/.openclaw/workspace/skills/pinchpass/
```

### Publish to npm

```bash
cd plugins/pinchpass
npm run build
npm publish
```
